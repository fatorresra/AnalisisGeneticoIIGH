Este caso de uso hace referencia a la solicitud de un 
analisis, la cancelación de este o una modificación.

Disparador: El usuario inicia del proceso de solicitud 
a traves de un disparador en la GUI o el sistema.

1. El usuario especifica el tipo de petición.
      A.
	1.1. El usuario solicita un analisis.
	1.2. El usuario especifica el tipo de analisis.
	Nota: Este tipo de analisis puede ser de las 
	capacidades de su caballo, en que puede destacar
	etc, tambien podria ser de los caballos que mas
	ganan de su granja, de las mejores razas, de las
	mejores familias de caballos, de las granjas con
	mejores caballos, de las competencias con peores
	caballos o al contrario...
	1.3. El usuario especifica un rango de precio para
	su analisis.
	1.4. El usuario da una descripción de su problema.
	1.5. El usuario pasa una lista de requerimientos.
	1.6. La petición se envia por medio de la plataforma.
      B.
	1.1. El usuario solicita modificar un analisis.
	1.2. El usuario especifica el tipo de modificación.
	Nota: Este tipo de modificación puede ser del tipo
	comercial (precio, fecha de entrega...) o puede ser
	relacionada con el analisis en si mismo (agregar 
	más requerimientos, eliminar requerimientos, entregar
	más datos para el analisis).
	1.3. El usuario da una breve descripción de la
	petición.
	1.4. El usuario agrega archivos. (opcional)
	Nota: En el perfil del usuario, se dispone de toda
	la info relacionada con su analisis, los requeri -
	mientos que el usuario envío, etc.
	1.5. La petición se envia por la plataforma.
      C.
	1.1. El usuario solicita cancelar un analisis.
	1.2. El usuario especifica el tipo de solicitud.
	      A.
		1.1. El cliente desea pausar su analisis.
		Nota: Esto podria pasar por motivos finan-
		cieros, el cliente no dispone de los recur-
		sos para completar el pago.
		1.2. El cliente especifica sus razones.
		1.3. El cliente detalla lo que desea que 
		ocurra con su analisis.
		Nota: Esto podria ser, pausar x tiempo y pagar
		una multa o pausar la entrega del analisis
		con una multa inferior.
		1.4. En cliente envia la solicitud.
		1.5. El sistema muestra al cliente las im-
		plicaciones de pausar el analisis.
		1.6. El sistema envia la petición.	
	     B. 
		1.1. El cliente decide cancelar su anali -
		sis.
		1.2. El cliente especifica el motivo.
		Nota: Si el motivo es relacionado con el
		incumplimiento de los acuerdos (el motivo
		debe ser de una lista que provee el sistema)
		no se cobra la penalización.
		1.3. El cliente da una descripcion detallada.
		1.4. El cliente envia la peticion.
		1.5. El sistema muestra al cliente las im-
		plicaciones de cancelar el analisis.
